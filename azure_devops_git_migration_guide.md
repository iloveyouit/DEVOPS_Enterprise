# Beginner's Guide: Connecting Your Local Code to Azure DevOps

Welcome! If you are new to Git and Azure DevOps, figuring out how to get your code from your laptop into the cloud can feel confusing.

This guide breaks down exactly what we just did to connect your `DEVOPS_Enterprise` code to Azure DevOps, step-by-step, so you can do it again for any future project!

---

## 🧠 Concepts to Understand First

- **Git**: The tool on your computer that tracks changes to your files (your "local repository").
- **Remote**: A cloud server where your code is backed up. You can have multiple remotes.
  - In our case, `origin` is the name of your GitHub remote.
  - We created a new remote called `azdo` for your Azure DevOps remote.
- **PAT (Personal Access Token)**: A secure, temporary password generated by Azure DevOps that gives your terminal permission to push code into your account.

---

## Step 1: Generate a Personal Access Token (PAT)

Before your terminal can talk to Azure DevOps, it needs a "key".

1. Log into your Azure DevOps portal (`https://dev.azure.com/143IT`).
2. In the top-right corner, click on the **User settings** icon (it looks like a little gear with a person inside it), then click **Personal access tokens**.
3. Click **+ New Token**.
4. Name your token (e.g., "Macbook Terminal Access").
5. Scroll down to the **Scopes** section. Find **Code** and select **Read & write**. _(This tells Azure to allow this password to push code)._
6. Click **Create** at the bottom.
7. **Important:** Copy the generated token string immediately! Azure will never show it to you again.

## Step 2: Create an Empty Repository in Azure DevOps

Your code needs a designated "bucket" to live inside Azure DevOps.

1. Go to your Azure DevOps project (e.g., **Head_Office**).
2. On the left menu, click **Repos** > **Files**.
3. At the very top center of the screen, click the repository dropdown (it likely displays the name of your default repo) and click **New repository**.
4. Give it a name (e.g., `DEVOPS_Enterprise`).
5. **CRITICAL STEP:** Make sure **"Add a README" is UNCHECKED**. If you check this box, Azure DevOps will create a file in the bucket, and Git will reject your upload because your local files and the cloud files will conflict.
6. Click **Create**.

## Step 3: Tell Your Computer Where Azure DevOps Is

Now, open your computer's terminal and navigate to the folder containing your code. We need to link your local code to the empty bucket you just made.

Run this command to add Azure DevOps as a new remote called `azdo`:

```bash
git remote add azdo https://PAT:YOUR_NEW_TOKEN_HERE@dev.azure.com/143IT/Head_Office/_git/YOUR_REPO_NAME
```

_Make sure to replace `YOUR_NEW_TOKEN_HERE` with the PAT you copied in Step 1, and `YOUR_REPO_NAME` with the name of the bucket you created in Step 2._

_(Note: We use `azdo` as the name instead of `origin` so your terminal doesn't get confused between GitHub and Azure DevOps!)_

## Step 4: Push Your Code to Azure

With the connection established, you simply upload your main code branch into Azure DevOps:

```bash
git push -u azdo main
```

- **`push`**: Means "upload my code".
- **`-u azdo`**: Tells your computer "Remember `azdo` as the default destination for this specific command."
- **`main`**: The name of your primary code branch.

---

## 🚀 How to Work Day-to-Day Now

Now that your setup is complete, you are linked to both GitHub and Azure DevOps!

When you make changes to a file and want to save it everywhere, you just run three commands:

1. **Save your changes to Git locally:**

   ```bash
   git add .
   git commit -m "Describe what changes you made"
   ```

2. **Send a copy to GitHub:**

   ```bash
   git push origin main
   ```

3. **Send a copy to Azure DevOps:**
   ```bash
   git push azdo main
   ```

You are now successfully using a dual-remote enterprise Git strategy!
